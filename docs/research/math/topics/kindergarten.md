# Math Research — Kindergarten / อายุ 5–6 ปี

_Last updated: 2026-06-03_  
_Status: Draft v1 for KidQuest curriculum design_

## Purpose

This document defines the first research-backed math learning layer for KidQuest children around age 5–6.

Target user:
- Thai child around kindergarten age
- Learns best through play, visual feedback, short sessions, and repetition
- KidQuest should feel like an RPG game, not a worksheet

---

## Evidence Base

### 1. Early math must be explicit, playful, and connected to daily life

WWC / IES “Teaching Math to Young Children” gives five recommendations for preschool, pre-K, and kindergarten math instruction:
1. Teach number and operations using a developmental progression.
2. Help children see math in everyday activities.
3. Use games, small groups, and teacher-led activities.
4. Teach geometry, patterns, measurement, and data.
5. Monitor progress and tailor instruction.

KidQuest implication:
- Do not only make random arithmetic drills.
- Each level should map to a micro-skill.
- Parent report should show skill growth, not only XP.

Source:
- Institute of Education Sciences / What Works Clearinghouse, “Teaching Math to Young Children”
- https://ies.ed.gov/ncee/wwc/PracticeGuide/18

---

### 2. Manipulatives and representations matter

EEF “Improving Mathematics in the Early Years and Key Stage 1” recommends using manipulatives and representations to develop understanding, but they must be used purposefully.

KidQuest implication:
- Use dots, blocks, fingers, eggs, rings, and objects as visual representations.
- Do not jump straight to abstract symbols.
- Every math question should have a visual option when the skill is new.

Source:
- Education Endowment Foundation, “Improving Mathematics in the Early Years and Key Stage 1”
- https://educationendowmentfoundation.org.uk/education-evidence/guidance-reports/early-maths

---

### 3. CPA: Concrete → Pictorial → Abstract

Singapore MOE describes Concrete-Pictorial-Abstract as a numeracy approach where children first use concrete objects, then pictorial representations, then abstract symbols.

KidQuest implication:
- In game terms:
  - Concrete = eggs, apples, blocks, creatures, rings
  - Pictorial = dot frames, ten frames, icons
  - Abstract = 3 + 2 = 5
- A level should not start with symbols only unless the child has already mastered the visual version.

Source:
- Singapore MOE NEL, “Using Concrete-Pictorial-Abstract (CPA) Approach”
- https://nel.moe.edu.sg/la/numeracy/how-can-you-do-it-/using-concrete-pictorial-abstract--cpa--approach/

## Core Design Rule

For age 5–6 math:

```text
Visual first
↓
Short challenge
↓
Immediate feedback
↓
Repeat with variation
↓
Mastery unlock
```

Avoid:
- Long explanation
- Passive watching
- Pure worksheet style
- Punishment for mistakes
- Too many new concepts in one level

## Recommended Skill Tree

### Strand A — Number Sense

Goal:
Child understands numbers as quantity, order, and comparison.

#### A1. Count objects 1–5

Lesson:
- Count eggs / stars / food pieces.
- Tap each object once.

Question types:
- “มีไข่กี่ฟอง?”
- Tap all eggs and choose number.
- Count objects that move slowly.

Game mechanic:
- Egg Nest Counting
- Child taps objects; each tap makes a sound and number label.

Mastery:
- 90% accuracy across 10 questions
- No double-counting in final 3 questions

---

#### A2. Count objects 1–10

Lesson:
- Extend counting to 10.
- Use stable order and one-to-one correspondence.

Question types:
- Count rings.
- Count creatures.
- Count food drops.

Game mechanic:
- Egg Run counting bonus
- After collecting rings, choose total.

Mastery:
- 85% accuracy across 12 questions
- Can count scattered objects, not only straight rows

---

#### A3. Match numeral to quantity 1–10

Lesson:
- Connect symbol “7” to seven objects.

Question types:
- Show numeral → choose matching group.
- Show group → choose numeral.

Game mechanic:
- Creature Feeding
- Feed the creature exactly 6 foods.

Mastery:
- 85% accuracy
- Fast recognition for 1–5, slower allowed for 6–10

---

#### A4. Compare more / less / same

Lesson:
- Compare two groups visually.

Question types:
- Which side has more?
- Which side has less?
- Are they the same?

Game mechanic:
- Egg Balance Scale
- Put groups on scale; choose more/less/same.

Mastery:
- 85% accuracy
- Includes equal groups

---

#### A5. Order numbers 1–10

Lesson:
- Understand before/after.

Question types:
- What comes after 6?
- Put 3, 5, 4 in order.
- Missing number: 4, 5, __

Game mechanic:
- Number Bridge
- Build bridge tiles in order.

Mastery:
- 85% accuracy
- Can solve missing number, not only recite sequence

---

### Strand B — Addition & Subtraction Foundations

Goal:
Child understands joining and taking away before formal calculation.

#### B1. Add within 5 with objects

Lesson:
- Joining two groups.

Question types:
- 2 eggs + 1 egg = ?
- Choose total group.

Game mechanic:
- Egg Merge
- Two nests combine; child counts total.

Mastery:
- 85% accuracy with visual support

---

#### B2. Add within 10 with objects

Lesson:
- Extend joining to 10.

Question types:
- 4 rings + 3 rings = ?
- Choose answer from 3 options.

Game mechanic:
- Ring Collector
- Creature collects two groups of rings.

Mastery:
- 80–85% accuracy
- Visual support still available

---

#### B3. Subtract within 5 with objects

Lesson:
- Taking away.

Question types:
- 5 foods, creature eats 2. How many left?
- Drag away objects, choose remaining.

Game mechanic:
- Hungry Creature
- Creature eats objects one by one.

Mastery:
- 85% accuracy with visual support

---

#### B4. Mixed add/subtract within 10

Lesson:
- Identify operation from story.

Question types:
- “ได้เพิ่ม” means add.
- “กินไป / หายไป” means subtract.

Game mechanic:
- Mini story quest
- Child sees short animated situation.

Mastery:
- 80% accuracy
- Must distinguish add vs subtract

---

### Strand C — Patterns & Logic

Goal:
Child sees structure, repetition, and prediction.

#### C1. AB patterns

Lesson:
- Red-blue-red-blue
- Egg-creature-egg-creature

Question types:
- What comes next?
- Complete pattern.

Game mechanic:
- Egg Garland
- Build pattern chain.

Mastery:
- 90% accuracy

---

#### C2. ABC patterns

Lesson:
- Three-part repeating patterns.

Question types:
- A-B-C-A-B-?
- Choose next object.

Game mechanic:
- Creature Parade

Mastery:
- 85% accuracy

---

#### C3. Sorting by one attribute

Lesson:
- Sort by color, shape, size.

Question types:
- Put all purple eggs together.
- Choose object that belongs.

Game mechanic:
- Egg Sorting Box

Mastery:
- 85% accuracy

---

### Strand D — Shape & Spatial Reasoning

Goal:
Child recognizes shapes and basic spatial language.

#### D1. Basic shapes

Lesson:
- Circle, square, triangle, rectangle.

Question types:
- Find the circle.
- Match shape to object.

Game mechanic:
- Shape Gate
- Creature passes through matching gate.

Mastery:
- 90% accuracy

---

#### D2. Position words

Lesson:
- บน / ล่าง / หน้า / หลัง / ข้างใน / ข้างนอก

Question types:
- Put egg inside basket.
- Which creature is under the tree?

Game mechanic:
- Egg Hide and Seek

Mastery:
- 85% accuracy

## Game Design Rules for Claude Code

When implementing math content:
1. Do not hardcode one-off question logic if a reusable question object can work.
2. Prefer visual models for new skills.
3. Keep one question under 10–20 seconds.
4. Use friendly hints instead of “wrong” messages.
5. Keep mastery based on accuracy EMA, not raw completion.
6. Do not modify `eggAlgorithm.js`.
