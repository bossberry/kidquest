## Live Systems

### Word Factory — "โรงงานประกอบคำ", isolated consonant+vowel blending mini-game, incl. compound vowels (2026-07-22, updated same day)
A fully separate mini-game (not a spec-doc item — a direct request) teaching a specific pre-reading gap: a child hasn't yet connected that a consonant + a vowel blend into a word's sound (vs. memorizing whole words). Deliberately isolated from every existing game system per explicit instruction — its own folder, own data, own progress storage, own screen. Reached via a new 🧩 "ประกอบคำ" `BottomNav` tab, never mixed into the existing egg/battle/room gameplay. **Same-day update**: real playtesting showed the single-vowel-only version had gotten too easy — the actual sticking point is compound vowels (ัว/เ-ีย/เ-ือ) — so a difficulty selector + 3 new compound-vowel levels were added directly (not gated behind the easy pool).
- **`src/wordfactory/data.js`** — pure data, no React. `VOWEL_DEFS` is the single source of truth for vowel positioning (`{type, before, after}` per vowel key) — `composeWord(consonant, vowelKey)` correctly builds every real word from it for all 4 shapes: single-after (า ิ ี ุ ู), single-before/leading (เ แ โ), compound wrap-after (ัว), and compound true-split (เ-ีย/เ-ือ, both a `before` AND `after`), with zero per-vowel-type branching anywhere else in the codebase. `getVowelLayout(vowelKey)` derives how the assembly stage should order its slots (`'trailing'`/`'leading'`/`'split'`) from the same before/after fields. `mkWord()` builds each `WORDS` entry (id/consonant/vowel/**vowelType**/**vowelParts**/displayWord/phoneticSteps/finalSound/imageKey/difficultyLevel — the spec's requested shape) from a vowel key + phoneticSteps, so `vowelType`/`vowelParts` are always derived, never hand-duplicated. `vowelGlyph(vowelKey)` (whole-vowel standalone display, e.g. a choice tile) is now just `composeWord('อ', vowelKey)` — generalizes cleanly to leading/compound vowels too (อา/อิ/.../เอ/แอ/โอ/อัว/เอีย/เอือ, all real Thai vowel-naming conventions). `vowelPartGlyph(fragment)` is the same อ-anchoring fix applied to ONE piece of a split-compound vowel (so a lone "ีย" fragment shown in its own slot doesn't float) — extends the original single-vowel rendering bugfix (see below) to the new split-slot case.
  - `EASY_WORD_IDS` (18 words: the original 6 สระอา + 12 more spanning ิ ี ุ ู เ แ โ — real words where they exist: มี/ตี/ปี/ปู/โต, honestly labeled `soundPractice: true` where no real standalone meaning exists for a 5-6 year old: มิ/มุ/มู/ปิ/ตุ/เก/แก) and `HARD_WORD_IDS` (10 words across ด่าน5 ัว: ตัว/หัว/มัว/วัว/บัว, ด่าน6 เอีย: เสีย/เมีย/เปีย, ด่าน7 เอือ: เสือ/เรือ) are the two pools the live game actually draws from. `LEVEL_1_WORD_IDS`/`LEVEL_2_WORD_IDS`/`LEVEL_5/6/7_WORD_IDS` are the same words grouped by the original ด่าน numbering, kept as named exports for reference. New consonants ห/ว/บ/ส/ร (needed for the compound-vowel word list) were added to `CONSONANT_SOUND` and a separate `ALL_CONSONANTS_HARD` distractor pool — kept OUT of the easy-mode `ALL_CONSONANTS` pool so easy mode's already-verified behavior never shows a never-before-seen consonant as a decoy.
  - `SWAP_PAIRS` (ด่าน4 "เปลี่ยนเพียงชิ้นเดียว") is still written but not wired into any UI — unchanged from the prior session, a future addition.
- **`src/wordfactory/storage.js`** — its own localStorage key `kq_wordfactory_stats_v1` (word/consonant/vowel miss counts, listen-replay count, stars, sessions completed). **Same-day addition**: `compoundVowelMisses` is now tracked SEPARATELY from `vowelMisses` (routed by the missed word's `vowelType`, passed in from `WordFactoryScreen.jsx`) — per direct instruction, confusing ัว with เอือ is a different signal than confusing ิ with ี, worth telling apart. Also added `sessionsCompletedByDifficulty: {easy, hard}`. Completely separate from `kq_state` — not real save progress, not protected by `hasRealProgress`/`validateState`, no migration handling needed (old saved blobs missing the new fields default-merge cleanly).
- **`src/wordfactory/WordFactoryScreen.jsx`** — single-file screen (intro w/ difficulty cards → 5-round build-a-word loop → reward). Intro now shows two big icon+color difficulty cards ("🟢 ง่าย" / "🟠 ยาก", minimal text per instruction) instead of one start button — tapping either starts that pool's session immediately, no gating. Each round: speak target word + show its picture → tap 1-of-4 consonant tiles (correct locks into an assembly slot with a chime; wrong bounces back gently + replays the target word, no ❌/buzzer) → same for the vowel (1-of-3, or all 3 in hard mode since only 3 compound vowels exist) → phonetic slow-readout then the full word via TTS → meaning caption → auto-advance. A within-session word that was missed can requeue once more (capped at `MAX_REQUEUE_PER_WORD = 2`), keeping the session at exactly 5 rounds. Reward screen gained a small "🔄 เปลี่ยนระดับ" link back to the difficulty select (in addition to replay-same-difficulty and go-home).
  - **The vowel is always ONE atomic tap, even for a 2-piece compound vowel** — the child taps a single tile showing the whole vowel (e.g. a tile reading "เอีย"), never 2 separate taps for the front/back pieces, per explicit instruction ("เด็กต้องเลือก 'สระผสม' เป็นชิ้นเดียว"). Only the ASSEMBLY STAGE display changes shape: the 2 fixed `[consonant][vowel]` slot boxes from the prior version are now a dynamically-ordered `assemblySlots` array built per round from `getVowelLayout()` — `'trailing'` keeps `[consonant, vowel]` (า ิ ี ุ ู, ัว), `'leading'` flips to `[vowel, consonant]` (เ แ โ — matches real script order), `'split'` renders 3 boxes `[vowelFront, consonant, vowelBack]` (เ-ีย/เ-ือ) that **both fill at the exact same instant** the single vowel tap resolves (not 2 taps) — live-verified in Chrome: tapping "เอือ" for เรือ instantly filled all 3 boxes as เ / ร / อือ, reading left-to-right exactly like real written Thai.
- **Reused shared infra (only, unchanged)**: `playTone`/`speakTh` (`src/lib/audio.js`) and `spawnConfetti` (`src/components/Toasts.jsx`'s already-globally-mounted `ConfettiLayer`). Zero dependency on `StateContext.jsx`, `state.js`, `curriculum.js`, or `questionBank.js`.
- **Entry point unchanged**: `src/components/BottomNav.jsx`'s 🧩 "ประกอบคำ" tab, `src/App.jsx`'s `wordfactory` route — no edits needed this update, the difficulty selector lives entirely inside `WordFactoryScreen.jsx`'s own intro phase.
- **Scope note**: this update followed a direct playtesting-driven instruction (not a spec doc) to add compound vowels as an immediately-unlockable "ยาก" mode — ด่าน4 "เปลี่ยนเพียงชิ้นเดียว" (swap mode) and the "ป๊าอ่านผิด" catch-the-mistake idea remain unbuilt, data/UI for swap mode (`SWAP_PAIRS`) still just scaffolded for later.
- **Placeholder, disclosed**: word illustrations are still emoji. A few compound-vowel words used the requested real words even where the concept is abstract for a 5-6 year old (มัว "blurry", เสีย "broken/lost", เมีย "spouse" — all explicitly named in the instruction's own word list) — captions kept short/neutral rather than over-explaining. TTS is still the existing browser `speakTh()`; the new compound spoken forms (เอีย/เอือ/อัว) are real, correctly-written Thai strings but weren't independently checked for pronunciation quality on a real device.

### Speaking & Reading-Aloud Mode (Learning Core Overhaul §1.4 — 2026-07-09)
- **`src/lib/speech.js`** — thin wrapper over the browser's Web Speech API (`SpeechRecognition`/`webkitSpeechRecognition`). `isSpeechAvailable()` feature-detects (false in any non-browser context, e.g. this project's Node test runner). `startListening(expected, {lang, timeoutMs})` returns `{ promise, stop }` — `stop()` calls the recognition's own graceful `.stop()` (not `.abort()`) so a "hold to talk" release still resolves with whatever was actually heard rather than discarding it; `listenForPhrase()` is a simple fire-and-forget convenience wrapper for callers that don't need early-stop (e.g. the dev harness below). Falls back to `{ heard:'', match:0, unavailable:true }` immediately when unsupported.
  - **Similarity**: `normalize()` lowercases, strips punctuation, and strips ONLY Thai tone marks (ไม้เอก/โท/ตรี/จัตวา) — deliberately NOT other Thai diacritics/vowel marks, since those change the word's actual identity (สระ mismatches are real misses, not just accent variance a young reader's imperfect tone shouldn't be penalized for). `levenshteinDistance()` + `similarity()` (standard `1 - distance/maxLength` ratio, 0-1). `classifyMatch()`: ≥0.7 `'correct'`, 0.4-0.7 `'almost'`, <0.4 `'incorrect'` (spec thresholds, boundaries inclusive on the lower side of each tier).
  - **Privacy** (code comment + this note, per instruction): recognition runs entirely inside the browser's own built-in speech API — this file never records, stores, or transmits raw audio anywhere. It only ever reads the short text transcript the browser's API returns, compares it locally, and discards it. Nothing is uploaded to any KidQuest server.
  - **Real edge case found+fixed during this session's own read-only review** (done while a temporary tool-availability gap blocked running build/test — used the wait productively rather than idly): `isSpeechAvailable()` only detects whether the `SpeechRecognition` constructor *exists*, not whether recognition actually *works*. iOS Safari specifically is known to expose the constructor while recognition silently fails (permission denied, no mic, service refusal) — without a fix, that would have meant every attempt just resolves to `heard:''` → similarity 0 → "incorrect" forever, with the child stuck and no path to the listening-mode fallback the spec explicitly calls for. Added `isPermissionError(code)` to distinguish a real permission/hardware/service failure from an ordinary "didn't catch that" (no-speech/aborted/network — worth a plain retry, not a fallback), threaded `errorCode` through `startListening`'s resolved result. Also fixed a related timing bug: the `timeoutMs` window now restarts from the real `onaudiostart` event (mic capture actually beginning) instead of counting from `.start()` — a first-time OS permission dialog can itself take several seconds, and counting the timeout from `.start()` would silently eat into the child's real speaking window before they'd even seen the prompt.
  - Verified via a committed regression suite (`src/lib/__tests__/speech.test.js`, 12 tests) covering `isSpeechAvailable`/`normalize`/`levenshteinDistance`/`similarity`/`classifyMatch`/`isPermissionError` — the actual `SpeechRecognition` wrapper itself can't be unit-tested (needs a real browser + microphone), which is exactly what the dev harness below is for.
- **`src/lib/readAloudWords.js`** — word/sentence pools per curriculum node, pulled directly from `wordPools.js` (NOT from `questionBank.js`'s `generateQuestion()`, since quiz-question shapes — sequence tiles, numpad answers, memory-card pairs — don't map onto "here's a word, say it out loud"). - **`src/games/minigames/ReadAloud.jsx`** ("อ่านให้ไข่ฟัง") — registered in the existing minigame pool (`MINIGAMES.readaloud` in `minigameLives.js`: unlock at creature battleLevel 4, 3 lives/day; `THEMES.readaloud` in `minigameUI.jsx`), same ready→playing→done phase pattern as the other 5 minigames. Match ≥0.7 → correct, coins/XP. 0.4-0.7 "almost" → retry once. <0.4 → TTS models the correct pronunciation, then one repeat attempt — max 2 tries per word either way, never traps the child. Coins = 3 × words fully matched (0-15 per round). **Listening-mode fallback** (`isSpeechAvailable()` false, e.g. some iOS Safari versions — OR a `speechBroken` flag set mid-game the moment a real `isPermissionError` fires, since recognition can't ever succeed for the rest of that round once permission is genuinely denied/unavailable, and the child shouldn't be scored "incorrect" for a systemic failure that isn't their fault; resets fresh at the start of each new round): skips the mic UI entirely — TTS speaks the target word, child taps 1 of 3 picture choices (target + 2 distractors from the same node pool) instead of speaking.

Lets a real tester pick Thai/English + a preset or custom expected phrase, tap to record, and see the heard transcript + similarity score + tier + `errorCode` (flagging permission/hardware issues specifically, with a note that the real app would fall back to listening mode there) logged on screen (newest first) — since this environment has no microphone to verify recognition quality directly. ### Persistence & Auth
- localStorage key `kq_state` — always-on, written every state change
- Supabase `eggs` table: `user_id, child_name, state_json, updated_at`
- **Mandatory login** — app fully blocked behind email/password auth; no guest mode
- **Onboarding gate** — new accounts must set name + schoolGrade + gender before app is accessible; `state.name === ''` is the trigger
- `state.schoolGrade` — parent-entered label string (e.g. "ป.1"), informational only, does not affect game progression
- `state.gender` — `'male' | 'female' | 'unspecified'`
- `state.stateVersion` — schema version (currently 1); `migrateStateShape()` deep-merges new nested fields on load
- `state.lastSavedAt` — timestamp used for cloud conflict resolution (replaces `rounds` counter). **`defaultState()` now sets this to `0`, not `Date.now()`** (fixed 2026-07-03 — see below): `0` means "never actually saved," so a genuinely blank/new local state can never masquerade as a recent real save. Only `saveState()` stamps a real `Date.now()`.
- `resolveSync(local, remote)` — single source of truth for cloud conflict resolution in `state.js`. Whole-object last-write-wins by `lastSavedAt` (tie → remote), but **before** that fallback, two additional safety nets run (added 2026-07-03, alongside the pre-existing `hasRemoteCreatures && !hasLocalCreatures` check): (1) `local.lastSavedAt === 0 && remote.lastSavedAt > 0` → remote wins outright; (2) `hasRealProgress(remote) && !hasRealProgress(local)` → remote wins — `hasRealProgress()` (new export in `state.js`) checks `hatchedEggs`/`xpThai+xpEng+xpMath`/`rounds`/`ownedItems`/`ownedRoomItems`/`grade`/`badges`, deliberately EXCLUDING `coins`/`happiness`/login fields, because those are the fields the first-mount maintenance dispatches (`DECAY_HAPPINESS`/`CHECK_DAILY_RESET`/`DAILY_LOGIN`/`ER_SAVE_SCORE`) can touch before the async cloud fetch resolves — using a maintenance-immune signal is what actually closes the cross-device data-loss race for a *returning* player (not just a literally-blank account), since those dispatches would otherwise inflate `local.lastSavedAt` before `resolveSync` ever runs. **Any reducer that mutates persisted state MUST stamp `lastSavedAt: Date.now()`**, otherwise a stale remote snapshot can win a later resolveSync and silently revert the change on reload (bugs fixed this way: `c74e83d` daily-login coins; 2026-07-01 cosmetics/room purchases). Reducers currently stamping it: `DAILY_LOGIN`, `BUY_ITEM`, `EQUIP_ITEM`, `BUY_ROOM_ITEM`, `PLACE_ROOM_ITEM`, `REMOVE_ROOM_ITEM`, and (per the 2026-07-01 full-sweep) all mutating reducers including every minigame `*_DEDUCT_LIFE` case.
- **Initial-sync gate** (`state.js`, 2026-07-03): a module-level `_initialSyncComplete` flag (`markInitialSyncComplete()`/`isInitialSyncComplete()`) blocks `saveState()` entirely — no localStorage write, no Supabase push — until the mount-time `loadState()` → `resolveSync()` → `dispatch(INIT)` chain has resolved (or safely failed; the chain now has a `.catch()` + `.finally(markInitialSyncComplete)` in `StateContext.jsx` so a network failure never permanently disables saving). This closes the actual race: without it, the very first render's `useEffect(() => saveState(state), [state])` could push an empty `defaultState()` to Supabase before the real cloud row had been fetched, overwriting real cross-device progress.
- **Blank-wipe guard** (`state.js`, 2026-07-03): both `saveState()` and `syncToSupabase()` independently refuse to persist/push a state whose `lastSavedAt` is falsy AND that carries no `hasRealProgress()` signal — belt-and-suspenders on top of the initial-sync gate, and the only guard that protects `syncToSupabase()`'s direct callers that bypass `saveState()` (the migration re-push inside `loadState()`, the `SIGNED_IN` auth listener).
- `SaveStatusIndicator.jsx` — fixed bottom-right badge showing saving/saved/error/offline via pub/sub (`onSaveStatusChange`)
- Logout: `supabase.auth.signOut()` + `localStorage.removeItem(KEY)` + `dispatch(INIT, defaultState())` — fully clears local state
- `_migrateBattleStats()` backfills all new fields for legacy eggs on load
- One-time migration flags: `_subjectLevelCalibrated` (recalibrate subjectLevels from levelMastery), `_itemsMigrated` (additive items→homeItems/battleItems merge), `_evoRechecked` (recheck all creature evoStage on load)
